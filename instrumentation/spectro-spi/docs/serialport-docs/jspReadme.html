Changes 4.9 - 5.0

Support for Sun PersonalJava (pJava) beta release on WinCE MIPS, SH3, SH4, ARM.

Support for EPOC32-ER5 on Psion ARM.

javax.comm.SerialPort: The internal event strings for <"Data Available", "Output Buffer Empty"> were changed to <"rxReadyCount", "txBufCount">

javax.comm.SerialPort: notifyOnOutputEmpty now checked for platform support.

javax.comm.SerialPort: SerialPortEventTask abort is faster.

Code that used to rely on getData to return a count of the data read at the time of the timeout should not be relied on....available

SerialPortLocal.java now does setTimeoutTx with a value based on txQueue size, and bitrate.

SerialConfig.getBitRateNumber supports non-indexed bitrates.

Native code fix for Windows bug that does not properly report txQueue/rxQueue sizes.

SerialPortLocal.isSupported now returns FALSE if supported string does not match.

SerialPortLocal.isSupported supports "getPortList".

SerialPortLocal.close checks isOpen instead of value of 'port' since EPOC uses port as handle.

get/set DisableActiveLink added to SerialConfig to support EPOC 'Link to desktop' control.

Modem now provides notifyBeforeModemEvent control.

SerialPortLocal static block and void constructor modified for Netscape trusted applets.

SerialPortLocal static block now executes featureSetup.

SerialPortLocal now declared to throw IOException. Requires jspComm 2.6

SerInputStream now returns -1 after Thread.interrupt().

Changes 4.3 - 4.9

SerialPortLocal.java now does System.exit(-1) when platform is not supported

java.comm.SerialPort modified to fix issue when using send-and-wait protocol with very small reply packets of the same size at fast bitrates. In some situations the thread that received the DATA_AVAIL event could read the rxQueue, post new data, which in turn would cause new data on the rxQueue in such a manner that the second reply was not 'seen' because the rxQueue had the same amount of data at the next check interval. SerialPortLocal.sendSinceTxBufCheck() was added to implement this this fix.

java.comm.SerialPort modified to fix port close issue when running on 'slow' machine (like WinNT on a P166) where the port is closed within 100 ms of it being opened.

Fixed native Win32 DLL to allow building EXEs with Visual Cafe when using port discovery (SerialPortLocal.getPortList).

JavaTerm now launches the GUI even if the native lib fails to load.

Fixed the enableReceiveTimeout feature in javax.comm.SerialPort, such that read() returns -1 when no data is available upon timeout.

SerialPortLocal.featureSetup changed "IRIX" to "Irix"

javax.comm.SerialPort implementation modified to prevent missing OUTPUT_BUFFER_EMPTY and DATA_AVAILABLE msgs in some unusual multi-threaded execution situations.

Enhanced javax.comm.SerialPort version of JavaTerm to deal with some timing issues on certain Win98 machines.

Support for Novell Netware 5.

SerialPortLocal.java now checks for "Windows 2000" as a valid option.

SerialPortLocal.read now limits the read length to the buffer size, previous to this change requesting to read more data than could fit in the passed buffer caused a fatal exception (at least on Win98).

SerialPortLocal.featureSetup modified for Unix platforms. SerialPortLocal.read(buf, offset, len) will now wait at Java level instead of native level for data to arrive. When using the timeout in the native code, Unix returns when any data is ready, not the requested amount. This meant that on Unix platforms the request would generally return when 1 byte was received no matter what the requested length.

Modified SnoopLoop so that getReadBuffer returns cumulative data properly.

JavaTerm modified to allow use on WinCE handheld PC (1/2 VGA)

TermTest example uses CTL-Q to exit instead of CTL-C so that it works on Netware via remote console.
